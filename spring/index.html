  • spring ： 给行业带来了春天

2、优点

  • spring是一个开源的免费框架（容器）
  • spring是一个轻量级的、非入侵式的框架。
  • 控制反转（IOC）、面向切面编程（AOP）
  • 支持事务的处理，对框架整合的支持。

总结一句话：*Spring就是一个轻量级的控制反转（IOC）和面向切面编程的框架*。

6、深入控制反转

所谓控制反转，可以理解为：获得依赖对象的方式反转了。

采用XML配置Bean的时候，Bean的定义和实现是分离的，而采用注解的方式可以把两者合为一体，Bean的定义直接以注解的形式定义在实现类中，，从而达到了零配置的目的。

控制反转是一种通过XML或者注解，并通过第三方去生产或者获取特定对象的方式。在Spring中实现控制反转的是IOC容器，其实现方法是依赖注入(Depende injection, DI).

7、使用xml配置数据

  1. 新建beans.xml，用容器来管理数据

  2. 在原有的dao层、业务层基础上，对每个接口，都写为bean，交给spring管理。

  3. 这样在需要用的时候，不用再通过new的方式获得对象

  4. 用上bean以后，对象由Spring来创建、管理、装配。

8、IOC容器创建对象

  1. 默认使用无参构造函数创建对象。

  2. 也有用有参构造函数的方法。

    1. 通过参数下标（参数过多时，可能搞错下标，有一定风险）

    2. 通过参数类型（当参数同类型时，不适用，局限性较大）

    3. 通过参数名（最推荐方法，因为参数不会重名）

总结：在配置文件加载（得到context）的时候，容器中管理的（所有）对象就已经初始化了，因此可以在后面要用的时候直接调用。

9、spring配置

1、别名——alias

通过alias标签，直接对bean一对一取别名。

2、Bean的配置

  • id——bean的唯一标识符（名字）
  • class——bean对应的对象的包名+类名
  • name——可以给这个bean取（多个）别名，比alias牛逼。
  • property——对应对象的属性
  • value——对应的值
  • construct-arg——构造函数的参数

3、Import

一般用于团队开发，可以将多个配置文件，导入到一个总的配置文件。

不同开发人员注册了不同的bean，可以将他们写出来的beans.xml合并到一个ApplicationContext.xml中，使用他们的bean中的对象。

【注意】时，用总的context，取import其他的（子）bean。

11、自动装配

  • 自动装配是使用spring满足bean依赖的一种方法。
  • spring会在应用上下文中为某个bean找到其依赖的bean。

三种装配机制：

  1. 在xml中显式配置
  2. 在Java中显式配置
  3. 隐式的bean发现机制和自动装配。【重点】

spring的自动装配需要从两个角度来实现或者说是两个操作：

  1. 组件扫描——spring会自动发现应用上下文中所创建的bean

  2. 自动装配——spring自动满足bean自建的依赖，也就是我们说的IOC/DI；

    （不推荐使用自动装配xml，而是使用注解。）

11.1、ByName自动装配

11.2、ByType自动装配

11.3、小结自动装配（狂）

byname：需要保证所有bean的id唯一，并且与对象属性的set方法的名字一致。

bytype：需要保证所有bean的class唯一，并且这个class要和属性的类型一致。

11.4、使用注解自动装配

使用Autowired注解

前提：

  1. 添加xml约束

  2. 开启属性注解支持

然后就可以给属性加上**@Autowired**注解，然后spring就会帮我们自动配置bean。

【注意】

  1. 这个注解会自动查看是满足byname还是bytype，如果name和type都不满足唯一性，那么需要和另一个注解——**@Qualifier（value=“”）**一起使用，此时Autowired以byname的形式运行。
  2. 有@Autowired的话，就可以省略set方法。
  3. Autowired优先使用ByType。

【题外话】

@Autowired有一个required属性，如果设置为false（默认为true），则说明对象可以为null。

否则的话没对象必须有值，不能为null。

使用@Resource注解

@Resource要比Autowired更好用一点。

它也可以做到，当有符合的名字时，优先使用名字符合的；当名字不符合，使用唯一符合的class的bean。

还可以对其指定要获取的bean——@Resource（value=“BeanName”）

11.5、小结

  • 都是用来自动装配的，都可以放在属性字段上。
  • @Auto wired优先使用ByType，要求这个对象必须存在。
  • @Resource优先使用ByName，不行的话就ByType，如果都找不到，就会报错。（还可以指定）

14、代理模式

1、静态代理

角色分析

  • 抽象角色：一般使用接口或者抽象类来实现

  • 真实角色：被代理的角色

  • 代理角色：代理真实角色，代理真实角色之后，我们一般会做一些附属操作

  • 客户：访问代理角色的人

优点

  • 可以使真实角色变得纯粹，不用实现一些公共的业务
  • 公共业务交给代理角色。实现了业务的分工
  • 公共业务改动时，方便管理。

缺点

每一个真实角色都会需要一个代理角色，代码量会翻倍，开发效率变低。（存疑）

优化代理模式

详见spring-08-proxy-demo02

2、动态代理

  • 动态代理和静态代理的角色一样
  • 动态代理的代理类是动态生成的，不是我们写好的
  • 动态代理分为两大类：基于接口的动态代理、基于类的动态代理
    • 基于接口——JDK动态代理
    • 基于类——cglib
    • Java字节码实现——javasist
  • 需要了解两个类
    • Proxy（代理）
    • Invocationhandler（调用处理程序）
  • 优点：
    • 涵盖静态代理的所有好处
    • 同时提高了代码的复用性，减少了代码量
    • 一个动态代理实现的就是一个接口，一般就是对应的一类业务。
    • 一个动态代理类可以代理多个类，只要是实现了同一个接口即可（存疑）

动态代理类的生成类

如何使用生成类得到动态代理类

“动态”的实质

以前我们要得到proxy，靠的是写一个proxy类，实现真实对象（可能还有多的功能要实现）

现在通过这个生成类，我们可以只传递真实对象作为接口，就能得到这个真实对象的代理角色。

区别就是如何的得到这个proxy对象，编写proxy类——传递真实对象接口

15、AOP

插一嘴:IOC的核心是工厂模式 AOP的核心是代理模式

1、 什么是AOP

在软件业，AOP为Aspect Oriented Programming的缩写，意为：面向切面编程，通过预编译方式和运行期间动态代理实现程序功能的统一维护的一种技术。AOP是OOP的延续，是软件开发中的一个热点，也是Spring框架中的一个重要内容，是函数式编程的一种衍生范型。利用AOP可以对业务逻辑的各个部分进行隔离，从而使得业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率。

2、AOP在Spring中的应用

提供声明式服务； 允许用户自定义切面；

即在不改变源码的条件下，增加新的功能。

18、声明式事务

1、回顾事务

  • 事务在项目开发过程中非常重要，涉及到数据的一致性的问题，不容马虎。
  • 事务管理是企业级应用程序开发中必备技术，用来确保数据的完整性和一致性。

事务就是把一系列的动作当作一个独立的工作单元，这些动作要么全部完成，要么全部不起作用。

事务的四个属性：

  1. 原子性
    • 事务由一系列动作组成，要么所有动作都完成了，要么所有动作都不起作用
  2. 一致性
    • 一旦事务完成，就要被提交。数据和资源处于一种满足业务规则的一致性状态中。
  3. 隔离性
    • 可能多个事务会同时处理相同的数据，因此每个事务都应该和其他事务隔离开，防止数据损坏。
  4. 持久性
    • 事务一旦完成，无论系统发生什么错误，结果都不会受到影响，通常情况下事务的结果被写到持久化存储器中。

2、Spring中的事务管理

  • 声明式事务：AOP（不改变源码）
  • 编程式事务：需要在代码中进行事物的管理——通过try、catch，事务回滚来实现。

事务是为了保证CRUD操作的同时进行，不会影响数据的一致性。
